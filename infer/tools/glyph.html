🔤 嵌入式字模 Alpha 提取工具

📐 验证 1：浏览器原生渲染 (带基线参考)
🖼️ 验证 2：JSON 数据反绘 (白底黑字 / 纯软件 Alpha 混合)
💡 提取与验证逻辑（紧凑裁剪 + 每字符 yOffset）
• 紧凑裁剪：位图精确裁剪到各字形的墨盒 actualBoundingBox，不再统一等高，最大限度节省存储。
• 每字符定位参数：xOffset = 位图左边缘相对笔位 (pen origin) 的偏移，可为负（如斜体左出血）；yOffset = 基线到位图顶部的距离（向上为正，即该字符 Ascent；可为负，如 _ 等整体位于基线下方的字符）；xAdvance = 笔位步进宽度。
• 嵌入式渲染公式：drawX = penX + xOffset，drawY = baselineY - yOffset；绘制后 penX += xAdvance。
• 全局行度量：lineHeight / baseline 仅用于换行与多行对齐，不参与单字定位。
• Alpha 还原：Canvas 混合后 Alpha 通道恒为 255，实际抗锯齿覆盖度已线性混合进 RGB。黑底白字下，直接读取 R 通道即可无损还原 0~255 覆盖度。
• 验证 1：原生渲染画框（青色=墨盒，品红刻度=笔位），核对物理边界与基线。
• 验证 2：完全基于 JSON 数据、按上述渲染公式逐像素混合。白底 `255` + 黑字 `0`，混合公式 `Result = 255 - Alpha`。若反绘结果与验证1一致，证明定位参数与数据格式完全正确。
• C 头文件：自包含 mcu_font.h。Alpha 量化为 4bpp（q=(v+16)/17），RLE token 流 + 13 字节定长稀疏索引（码点升序、二分查找），全部 const uint8_t 字节流、小端位移装配，架构无关、无浮点。接口 mcu_font_get_glyph(cp, alpha, &w, &h, &xo, &yo, &adv)（xo/yo 为 int8_t*）：alpha=NULL 仅取元数据；传入缓冲区则解码字模（每像素 1 字节，值 0~15）。配套测试见 test_mcu_font.c。
